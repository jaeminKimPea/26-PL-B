## AI 활용 윤리
### 핵심 요약:
- **AI는 '정답 자판기'가 아닌 '성장의 도구'**
- [**과정 공유**]
  - AI 사용 및 프롬프트 내역을
투명하게 공개
- [**주도권 확보**]
결과물 검증과 최종 책임은
오직 나의 몫
- [**본질 집중**]
'결과'가 아닌 '원리(Why)'를
묻는 습관 형성
>부가설명
- **3대 핵심 원칙**
  1. **정보의 투명성** - 숨기지 않고 명확하게 밝히기
   - AI 활용 여부 밝히기
   - 활용한 프롬프트 기록
   - AI답변 바탕 수정 및 보완 명시
  2. **저작권 및 책임** - 최종 책임은 사용자 본인에게 존재
   - 생성물의 저작권 위반 여부 확인 의무
   - 생성된 코드의 논리적 결함 직접 검증
   - 최종적으로 결함에 대한 모든 결과 인지
  3. **기술적 종속 경계** - 무비판적 수용 지양 및 주도권 확보
   - 무비판적 수용 태도 지양
   - 어떻게 작동하는 가 질문 습관
   - 직접 논리 검증 해답 도출

## IDE vs TERMINAL
- IDE - 통합스튜디오
Visual Studio Code, Cursor IDE,
PyCharm + AI 플러그인
    - 구조 시각화
    - 시각적 디버깅 및
브레이크포인트 제어
    - 인라인 코드 설명 및
리팩토링 지원
    - 코드 자동완성 및 생성
(Copilot, GPT 기반)

- TERMINAL
    - 텍스트 기반의 코드 생성 및
즉시 실행
    - 기존 시스템 도구와의 강력한
파이프라인 결합
    - 자동화 스크립트 작성 및 서버 환경 활용 극대화

## IDE 형
- #### Cursor AI
    - VS Code 기반 확장형 개발 환경
 프로젝트 전체 문맥을 이해하는 LLM 통합

    - Cursor가 하는 일

        - 코드 생성 및 설명

        - 버그 분석 및 리펙토링 제안

        - 테스트 코드 생성

        - 문서화 자동 생성
    - Inline Edit (즉석 코드 생성): Ctrl + K

    - Al Chat (코드 대화 및 튜터링): Ctrl + L

    - @ Symbols (Context Targeting)
        - @ 기호의 역할: 특정 파일이나 문서를 AI에게 '참조'로 전달
    - Cursor의 처리: 해당 파일의 내용을 문맥(Context)에 포함하여 답변 생성
- #### Antigravity
개발자의 생산성을 위한 차세대 AI 코딩 어시스턴
스

- AI 기반 코드 생성 및 수정 도구

- 코드 설명, 리펙토링, 테스트 생성 지원

- 대화형 인터페이스 기반 개발 지원

- 정체성 - 일을 대신하는 로봇 사원

- 작동 단위 - 기능 /태스크

- 권한 - 파일 생성 + 터미널 + 브라우징

- 사용자 역할 - 감독관 (목적지 설정 및 승인)
